## 1. 项目概述

### 1.1 项目定义

**项目名称：** 日本都道府県 宜居度指数 (Japan Livability Index)

**一句话描述：** 基于日本政府公开统计数据，通过 15 项指标对 47 个都道府县进行多维度宜居度评分排名，并以交互式地图可视化呈现。

**核心叙事：** "东京并非生活最舒适的地方" —— 用数据打破认知偏差。

### 1.2 项目亮点

- **数据驱动：** 全部数据来自日本政府 e-Stat 公开统计 API，可溯源、可复现
- **交互式可视化：** D3.js 制作的日本地图，支持点击查看详情和权重动态调节
- **多维评分模型：** 5 大维度 × 15 项指标（14 项采集 + 1 项派生）
- **权重预设：** 4 种生活场景预设（默认/家庭/职业/养老）+ 自定义滑块
- **内容策略：** 面向小红书在日华人群体的数据故事

### 1.3 目标受众

| 受众 | 关注点 |
|-----|-------|
| GitHub 开发者 | 完整的数据工程管线、代码质量、文档 |
| 在日华人（小红书） | 排名结果、东京 vs 其他县的对比 |
| 数据分析爱好者 | 评分方法论、权重设计的合理性 |

## 3. 评分模型

### 3.1 指标体系（权威映射表）

以下映射来自 `docs/weight_rationale.md`，为本项目的**唯一权威定义**。所有代码和文档必须与此一致。

| 维度（日文原名） | 默认权重 | 包含指标 | 反转? |
|----------------|---------|---------|-------|
| **経済力** | 25% | 県民所得_一人当たり | 正向 |
| | | 完全失業率 | **反转**（低=好） |
| | | 有効求人倍率 | 正向 |
| **生活利便性** | 30% | 持ち家比率 | 正向 |
| | | 住宅延べ面積 | 正向 |
| | | 家賃_1畳当たり | **反转**（低=好） |
| | | 収入対家賃比（派生） | **反転**（低=好） |
| **環境快適度** | 20% | 年間日照時間 | 正向 |
| | | 年間降水量 | 正向（注1） |
| **社会安全度** | 10% | 犯罪発生率 | **反转**（低=好） |
| | | 交通事故発生率 | **反转**（低=好） |
| **将来性** | 15% | 人口増減率 | 正向 |
| | | 転入超過率 | 正向 |
| | | 高齢化率 | **反转**（低=好） |

**注1：** 年間降水量目前为正向处理（多=好）。如需调整方向，需同步修改 3 处：(1) `merge_all.py` 的 `INVERSE_INDICATORS` 列表，(2) 本文档映射表，(3) `docs/weight_rationale.md`。

**未分配指标（采集但未纳入评分，Post-MVP 处理）：**
- `病院数_人口10万対`
- `医師数_人口10万対`

> 这两项已被采集但当前不参与评分。Post-MVP 考虑纳入「生活利便性」维度。当前指标总数为 15（14 采集 + 1 派生），不含这两项。

**完整反转指标列表（代码中的 `INVERSE_INDICATORS`）：**
```python
INVERSE_INDICATORS = [
    "完全失業率",
    "犯罪発生率",
    "交通事故発生率",
    "高齢化率",
    "家賃_1畳当たり",
    "収入対家賃比",
]
```

### 3.2 评分算法

```
1. 采集:   e-Stat API → 14 项原始指标 (estat_collector.py)
2. 派生:   追加 収入対家賃比（第15项）
3. 归一化+反转: 单步完成 —
     正向指标: (x - min) / (max - min) × 100
     反转指标: (1 - (x - min) / (max - min)) × 100
   （代码在 merge_all.py normalize() 函数中一次性处理）
4. 维度聚合: 同一维度内指标取均值 → dim_経済力, dim_生活利便性, ...
5. 总分:   Σ(维度分数 × 权重) / Σ(权重)  ← 权重自动归一化
6. 排名:   按总分降序
```

### 3.3 权重预设

| 预设 | 経済力 | 生活利便性 | 環境快適度 | 社会安全度 | 将来性 | 场景 |
|------|-------|----------|----------|----------|-------|------|
| デフォルト | 25% | 30% | 20% | 10% | 15% | 通用 |
| 家庭向け | 15% | 35% | 20% | 20% | 10% | 有孩家庭 |
| 単身経済向け | 40% | 30% | 10% | 10% | 10% | 职业人士 |
| 退休向け | 10% | 25% | 35% | 20% | 10% | 退休移居 |

**自定义滑块行为：** 5 个滑块允许用户自由调节，总和不要求严格等于 100%。`app.js` 中 `recomputeScores()` 会自动归一化权重（除以总和），确保计算正确。UI 会显示当前总和，总和 != 100% 时显示红色警告。

#### 关键实现：维度聚合（任务 0.4）

这是目前数据管线中**缺失的关键步骤**。`merge_all.py` 当前只输出每个指标的归一化分数，但 `app.js` 期望读取 `dim_${维度名}` 格式的聚合分数。

需要在 `merge_all.py` 的 `run()` 函数中添加维度聚合逻辑：

```python
# 维度 → 指标映射（与 docs/weight_rationale.md 一致）
DIM_INDICATORS = {
    "経済力":     ["県民所得_一人当たり", "完全失業率", "有効求人倍率"],
    "生活利便性": ["持ち家比率", "住宅延べ面積", "家賃_1畳当たり", "収入対家賃比"],
    "環境快適度": ["年間日照時間", "年間降水量"],
    "社会安全度": ["犯罪発生率", "交通事故発生率"],
    "将来性":     ["人口増減率", "転入超過率", "高齢化率"],
}

# 归一化后，计算各维度均值
for dim_name, indicators in DIM_INDICATORS.items():
    cols = [c for c in indicators if c in norm.columns]
    if cols:
        norm[f"dim_{dim_name}"] = norm[cols].mean(axis=1).round(1)
```

#### 待确认的 API 编码

| 指标 | statsDataId | cdCat01 | 状态 |
|-----|-------------|---------|------|
| 家賃_1畳当たり | 0000010208 | H2130 | ※要確認 |
| 病院数_人口10万対 | 0000010209 | I5100 | ※要確認 |
| 医師数_人口10万対 | 0000010209 | I5210 | ※要確認 |
| 年間日照時間 | 0000010202 | B1103 | ※要確認 |
| 年間降水量 | 0000010202 | B1102 | ※要確認 |

**降级策略：** 若某编码查询无结果，该指标标记为缺失。评分模型中，缺失指标不参与该维度的均值计算（`mean(axis=1)` 自动忽略 NaN）。若某维度所有指标缺失，该维度得分设为 50（中位数）。

#### 已知风险

- API 有限流机制（1秒/请求），采集 14 项约需 15-30 分钟
- 部分指标可能不存在最新年份数据
- 「※要確認」编码错误可能导致 5/14 项指标缺失

**视觉层级（用户看到的顺序）：**
1. 警告 banner（样本数据时最先看到）
2. 地图（视觉主角，占 60%+ 屏幕）
3. 排名列表（目光右移到侧栏）
4. 权重面板（工具，不是内容）
5. 详情面板 + 雷达图（按需展开）
6. 操作按钮（Header 右侧，辅助功能）

### 9.2 新功能 UI 规范（CEO 审查批准的 5 项）

**样本数据警告 banner：**
- 位置：页面最顶部，Header 上方
- 样式：`background: rgba(var(--accent-rgb), 0.15); color: var(--accent);` 全宽
- 内容：「⚠ サンプルデータを表示中 — 実データは data/livability_scores.json を配置してください」
- 关闭按钮：右侧 [×]，点击后用 `sessionStorage` 记住状态，同一会话不再显示

**雷达图对比（详情面板内）：**
- 触发：详情面板下方「↔️ 比較対象を選択」下拉菜单，选择第 2 个县后自动切换到雷达图视图
- 切换：条形图/雷达图用 tab 按钮切换（「バー」/「レーダー」）
- 雷达图颜色：第 1 个县用 DIM_COLORS 各维度色，第 2 个县用同色半透明 (0.3)
- 动画：切换时 200ms 过渡动画
- 空状态（只选了 1 个县）：显示单县雷达图 + 「もう1つ選択して比較」

**截图导出：**
- 按钮：Header 右侧图标按钮 📸（hover 显示 tooltip「スクリーンショット」）
- 范围：截取整个应用界面（header + 地图 + 侧栏）
- 实现：html2canvas
- Loading：按钮禁用 + 内置 spinner + 「画像を生成中...」
- 成功：触发浏览器下载 + 底部 Toast「✅ 画像をダウンロードしました」(2s)
- 错误：Toast「❌ スクリーンショットに失敗しました」(3s)

**CSV 下载：**
- 按钮：Header 右侧图标按钮 📋（hover tooltip「CSVダウンロード」）
- 内容：当前权重下的排名数据（pref_name, rank, total_score, dim_* × 5）
- 触发：点击后瞬时下载，Toast「✅ CSVダウンロード開始」(2s)

**URL 参数分享：**
- 按钮：Header 右侧图标按钮 🔗（hover tooltip「URLをコピー」）
- 编码内容：权重配置 + 选中的都道府県
- 格式：`?w=25,30,20,10,15&p=13`（w=5维度权重, p=pref_code）
- 成功：复制到剪贴板 + Toast「✅ URLをコピーしました」(2s)
- 页面加载时：检测 URL 参数，自动还原权重和选中状态

### 9.4 Toast 组件规范

- 位置：页面底部居中
- 样式：`background: var(--surface); border: 1px solid var(--border); border-radius: 8px; padding: 0.6rem 1rem;`
- 动画：slide-up 200ms 进入，fade-out 200ms 退出
- 持续：成功 2s，错误 3s
- ARIA：`role="status" aria-live="polite"`

### 9.5 基础无障碍（a11y）

- `:focus-visible` outline: `2px solid var(--blue)` 所有交互元素
- 排名列表：`role="list"`，项为 `role="listitem"`
- 地图 SVG：`aria-label="日本都道府県地図"`
- 每个县 path：`aria-label="{pref_name} スコア {score}"`
- Toast：`aria-live="polite"` + `role="status"`
- 滑块：`aria-label="{dim_name}の重み"` + `aria-valuetext="{value}%"`
- 对比度：`--muted` 需调亮到 `#9ca3af`（WCAG AA 4.5:1）
